Overview

The proliferation of mobile devices with video and sensor recording capabilities has created new opportunities for multimodal sports analysis. However, there is a lack of public datasets featuring synchronized multi-view recordings of live sports events captured directly in the field by multiple smartphones.

To address this gap, we introduce MUVS — Multimodal Multi-view User-Generated Sports Dataset — a dataset of multi-view video and inertial sensor data collected simultaneously from multiple Android smartphones during live sports events, enabling research in camera selection, viewpoint ranking, and video mashup generation.

Each smartphone captured video alongside 18 onboard sensor channels — including accelerometer, gyroscope, magnetometer, orientation, ambient light, pressure, proximity, and GPS — providing a rich multimodal signal for mobile sensing and activity recognition research. The dataset covers Basketball, Soccer, Hockey, and Volleyball events.

Data Details

MUVS is organized into overlap periods — time intervals where at least two cameras were simultaneously active. Each period is segmented into aligned 3-second fragments, enabling segment-level comparison and selection across viewpoints.

Video

Period-level videos encoded in H.264 (MP4, CRF 23), hosted on Google Cloud Storage. Faces are anonymized using Gaussian blur (YuNet detector) prior to public release.

Sensor Data

Fragment-level sensor measurements from 18 channels: accelerometer (x/y/z), gyroscope (x/y/z), magnetometer (x/y/z), orientation (x/y/z), ambient light, pressure, proximity, and GPS (latitude, longitude, speed).

Annotations

Fragment-level camera selection labels collected via a dedicated labeling tool. For each 3-second fragment, an expert annotator selected the best camera among all simultaneously active viewpoints.

Splits

The dataset is provided in train and test splits, with events distributed across both splits to enable cross-sport and cross-venue evaluation.

Download

MUVS is distributed in two parts: metadata and annotations are archived on Zenodo, while period videos are hosted on Google Cloud Storage and downloaded via the provided script.

1
Download the Zenodo package

Contains fragment-level metadata (sensor data, timestamps), camera selection annotations, the video download script, and this README.

2
Download the videos from GCS

Run the included download_videos.py script to fetch period videos from Google Cloud Storage. No credentials required — the bucket is publicly accessible.

Download on Zenodo

Downloading Videos

After downloading the Zenodo package, run the following commands from inside the package directory:

# Install dependencies
pip install -r requirements.txt

# Download all videos (~164.9 GB)
python download_videos.py

# Download to a custom directory
python download_videos.py --out /path/to/videos

# Download only specific events
python download_videos.py --events naperville_kids_match_20240608

# Preview without downloading
python download_videos.py --dry_run

The script is resumable: already downloaded files are skipped on re-run.
